1 قواعد ومواصفات النص العربي في اللغة Arabic Text Conventions

تمت تزويد لغة الأردوينو العربية بفلتر معالجة نصوص ذكي يعمل صامتاً قبل تصريف الكود (في مرحلة تسوية النص)، ويتيح هذا الفلتر مرونة تامة أثناء كتابة الكود وفقاً للقواعد التالية:

التشكيل

يمكن كتابة الكلمات مشكّلة كلياً أو جزئياً أو بدون تشكيل، وسيتعرف عليها المترجم كرموز متطابقة تماماً.

مثال: مُتَغَيِّر مطابقة تماماً لـ متغير في نظر المترجم.

التطويل (الكشيدة)

يمكن مد الكلمات بصرياً لتنسيق الكود باستخدام حرف الكشيدة ـ دون أي أثر على الدلالة.

مثال: طـــالما مطابقة تماماً لـ طالما.

توحيد الهمزات

يقوم المترجم بدمج وتوحيد كافة أشكال الهمزات (أ أو إ أو آ) وإرجاعها لألف عادية مهملة (ا) لتجنب أخطاء لوحة المفاتيح الشائعة.

مثال: كتابة إعداد أو اعداد كلاهما صحيح ويُرجعان لـ اعداد.
تنبيه: يجب كتابة كافة الكلمات المفتاحية في الكود بألف عادية مهملة لضمان التوافق مع مرحلة معالجة النصوص.

3 أنواع البيانات والأرقام الثنائية Data Types & Literals

يدعم المترجم أنواع البيانات التالية ويتحقق منها بصرامة في مرحلة التحليل الدلالي.

3.1 — صحيح / رقم (IntType)

مخصص لحفظ الأعداد الصحيحة الموجبة والسالبة، يُخصَّص له 32 بتاً برمجياً عتادياً. 

متغير اليد : صحيح = ١٣ ؛

3.2 — عشري / كسري (FloatType)

مخصص لحفظ الأعداد ذات الفاصلة العشرية لضمان دقة العمليات الحسابية. 

متغير الحرارة : عشري = ٢٣.٥ ؛

3.3 — نص (StringType)

مخصص لحفظ الجمل البرمجية المحاطة بعلامات الاقتباس المزدوجة. 

متغير رسالة : نص = "مرحبا بكم" ؛

3.4 — منطقي (BoolType)

يستقبل قيمتين فقط هما صح أو غلط للتحكم بمسارات القرار في الكود. 

متغير الحالة : منطقي = صح ؛

3.5 — الأرقام الثنائية المبتكرة (BIN_NUMBER)

ميزة متقدمة تتيح برمجة البتات والمنافذ عتادياً بكتابة حرف ث تتبعه سلسلة البتات بالأرقام الشرقية أو المشرقية. يقوم المترجم تلقائياً بتحويلها إلى قيمتها في النظام العشري. 

منفذ_ب = ث١١١١٠١١١ ؛  // يعادل القيمة 247 في النظام العشري

4 المعاملات والعمليات Operators & Expressions

يعتمد المترجم نظام أولويات صارم للعمليات لتفادي الغموض النحوي، وتُطبَّق قواعد الفحص الدلالي على كل معامل وفق نوع المعاملات.

4.2 — معاملات الإسناد المبسطة (Assignment & Desugaring)

5 جمل التحكم والشرط والتكرار Control Flow

تخضع جمل التحكم لشروط صارمة لحماية العتاد، وتوفر نطاق رؤية محمياً للذاكرة (Block Scope).

5.1 — جملة الشرط لو / اذا

تتحقق من شرط معين وتنفذ الكتلة المرافقة له إذا تحقق، مع دعم فرع والا البديل. 

لو ( العداد == ١٠ ) {
    العداد = ٠ ؛
} والا {
    العداد = العداد + ١ ؛
}
قاعدة دلالية: يجب أن يكون التعبير بين قوسي لو من نوع منطقي (BOOL_TYPE) حصراً، وإلا يرفض المحلل الدلالي الكود.

5.2 — حلقة التكرار طالما

تكرر تنفيذ الكود المحاط بالأقواس طالما أن الشرط المنطقي محقق. 

طالما ( س < ١٠ ) {
    س++ ؛
}
قواعد اقطع واستمر: يُسمح باستخدامهما حصراً داخل كتل طالما. يؤدي استخدام أي منهما خارج حلقة تكرار إلى خطأ دلالي فوري: "لا يمكن استخدام أمر 'اقطع' خارج حلقات التكرار (طالما)".

6 الكيانات والمكتبات المدمجة للأردوينو Arduino Standard Library

6.1 — دوال التحكم بالمنافذ والإشارات الرقمية

الدالةالوصفمثال

الدالةالوصفمثال

8 أمثلة برمجية كاملة مشروحة Complete Code Samples

المثال الأول — وميض الليد (Blink)

يهيئ المنفذ رقم 13 المدمج بالأردوينو كمخرج، ثم يشغّل الليد ويطفئه بصورة دورية كل ثانية. 

// برنامج وميض الليد باللغة العربية للأردوينو
متغير ليد : صحيح = ١٣ ؛

مهمة اعداد() : فارغ {
    وضع_الطرف(ليد، مخرج) ؛         // تهيئة المنفذ كمخرج رقمي
}

مهمة تكرار() : فارغ {
    اكتب_رقمي(ليد، عالي) ؛          // إضاءة الليد
    انتظر(١٠٠٠) ؛                   // انتظر ثانية كاملة
    اكتب_رقمي(ليد، منخفض) ؛         // إطفاء الليد
    انتظر(١٠٠٠) ؛                   // انتظر ثانية كاملة
}

المثال الثاني — قراءة مستشعر تماثلي وبث القيم تسلسلياً

يستورد مكتبة السيريال، ويقرأ جهداً تماثلياً متغيراً من المنفذ A0، ويرسل القيمة إلى الحاسوب باستمرار. 

استيراد "سيريال" ؛                  // استدعاء مكتبة الاتصال التسلسلي

متغير منفذ_المستشعر : صحيح = ٠ ؛   // المنفذ التماثلي A0

مهمة اعداد() : فارغ {
    سيريال_ابدا(٩٦٠٠) ؛            // تفعيل الاتصال بسرعة 9600 baud
}

مهمة تكرار() : فارغ {
    متغير القيمة : صحيح = ٠ ؛
    القيمة = اقرا_تناظري(منفذ_المستشعر) ؛

    سيريال_اطبع("قيمة المستشعر:") ؛
    سيريال_اطبع(القيمة) ؛          // بث القيمة لشاشة الحاسوب
    انتظر(٥٠٠) ؛                   // انتظر نصف ثانية بين القراءات
}

المثال الثالث — الحماية من القسمة على صفر (Divide-by-Zero Guard)

يوضح آلية حماية المترجم للمعالج من الانهيار عند محاولة القسمة على صفر؛ حيث يولد المترجم تلقائياً كود فحص يتحقق من قيمة المقسوم عليه قبل إجراء العملية، وعند اكتشاف الصفر يستدعي دالة الذعر panic_div_zero() التي تبدأ بوميض سريع لليد رقم 13 تحذيراً من الانهيار الرياضي. 

متغير س : صحيح = ١٠ ؛
متغير ص : صحيح = ٠ ؛

مهمة اعداد() : فارغ {
    // تهيئة عادية
}

مهمة تكرار() : فارغ {
    متغير النتيجة : صحيح = ٠ ؛

    // المترجم يولد كود فحص تلقائي قبل هذه العملية:
    // إذا كان ص == 0  ←  استدعاء panic_div_zero() فوراً
    النتيجة = س / ص ؛
}
